# 🧠 Cognitive AI Agent — Hybrid Architecture v1.9

> **FEOFALLS Team** | Arc - Builder | OpenClaw Native Integration | April 2026

Production-grade architecture specification for autonomous **Solo AI Agents** running on the **OpenClaw Gateway**. Defines memory governance, tool management, cognitive retrieval, and system integrity protocols.

---

## 🏗️ What This Is

The **OpenClaw Hybrid Architecture v1.9** is a complete blueprint for building autonomous, self-governing AI agents. It is *not* a code library — it is a **specification and governance framework** that an agent loads as its operational constitution.

Key design pillars:
- **Single Source of Truth** — All agent memory lives in Markdown files, not databases
- **Deterministic Memory** — Node IDs are generated by script, never by LLM (prevents hallucination)
- **Supply Chain Hardened** — SHA-256 script integrity + dependency pinning + isolation enforcement
- **Token-Efficient** — Dual-prompt bootstrap, heartbeat economy, session budget tracking
- **Self-Auditing** — Weekly parity audit detects spec-vs-implementation drift before it becomes a bug

## 🔑 Key Architecture Highlights

### Memory Layer Hierarchy
```
0_CONSTITUTION  (IMMUTABLE) ──► governs ──► 1_IDENTITY (PINNED)
                                                    │
                                            2_COGNITIVE/HEURISTICS
                                                    │
                                            3_LEARNING (Episodic → Semantic)
                                                    │
                                  4_OPERATIONAL_DATA/rag/ (ChromaDB + scripts)
                                                    │
                                            5_WORKING_STATE (runtime)
```

### Core Mechanisms
| Feature | Function |
|---------|----------|
| **Tool Harness Layer** | Centralized tool registry driving execution wrappers across the agent. |
| **Supply Chain Defense** | SHA-256 integrity + dependency pinning + docker/sandbox isolation protocols. |
| **Hard BLOCK enforcement** | Unregistered or unauthorized tools are instantly blocked. |
| **Parity Audit** | Weekly automated spec-vs-implementation discrepancy tracking. |
| **Session Intelligence** | Operational token budget tracking scaling natively with transcript compaction. |
| **`[OBSERVATION]` memory type** | Discrete block structure for recording real-world Creator feedback. |
| **Structured Output Retry** | Engages auto-repair logic handling malformed outputs (max 2 attempts). |

### Automation (4 Native OpenClaw Cron Jobs)
| Job | Schedule | Purpose |
|-----|----------|---------|
| Nightly Memory Indexer | `0 3 * * *` | Lifecycle sweep, salience update, auto-consolidation |

## 🛡️ Supply Chain Security Rationale

This architecture was hardened in response to two real-world supply chain attacks analyzed by the FEOFALLS team:
- **Axios npm compromise** (2026-03-31) — malicious package with self-deletion post-install
- **LiteLLM PyPI infostealer** (2026-03-24) — PyPI package with self-deletion mechanism

Standard virus scanning **cannot** detect self-deleting malware. The defense is architectural:
1. Pin all dependencies to exact versions with SHA-256 hashes
2. Run tools with external dependencies in Docker/Sandbox (never native)
3. Verify SHA-256 of every script before execution

## 🚀 How to Apply This Spec on OpenClaw

### Prerequisites
- OpenClaw Gateway v2026.3+ with Solo Agent enabled
- Python 3.10+ (for RAG scripts)
- ChromaDB (for vector storage)

### Step-by-Step

1. **Clone this repo** and scaffold the `memory/` directory structure as defined in `ARCHITECTURE_HYBRID_v1.9.md` Section II.

2. **Configure your Identity** — Edit `0_CONSTITUTION/` (axioms, boundaries) and `1_IDENTITY/` (agent profile) to match your agent's purpose.

3. **Deploy RAG scripts** — Place `write_helper.py`, `query.py`, `nightly_indexer.py` into `memory/4_OPERATIONAL_DATA/rag/`. Implement them following the specs in `TOOL_HARNESS_LAYER.md`.

4. **Set up the Tool Registry** — Create `tool_registry.json` following the schema in `TOOL_HARNESS_LAYER.md` Section 2. Register every script with its SHA-256 hash.

5. **Register Cron Jobs** — Run `cron_setup.py` or manually register 4 native OpenClaw cron jobs as listed in `AGENT_OPERATIONAL_GUIDE_v1.9.md` Section 9.

6. **First Boot** — Load the bootstrap sequence (Runtime Mode) from `AGENT_OPERATIONAL_GUIDE_v1.9.md` Section 1A. Write your first memory entry using `write_helper.py`.

7. **Verify** — Run a full Parity Audit (`parity_audit.py --full`) following the procedure in `PARITY_AUDIT.md` to confirm zero gaps between spec and implementation.
